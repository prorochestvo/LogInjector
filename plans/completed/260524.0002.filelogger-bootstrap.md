# Task Breakdown — File Logger Bootstrap Helpers

## Overview

Three downstream consumers of `github.com/prorochestvo/loginjector` each carry a
near-identical, copy-pasted `internal/logger.go`. Its core is a constructor along the
lines of:

```go
func NewLogger(logsDir, name string, printerMinLevel loginjector.LogLevel) (*loginjector.Logger, error)
```

that does four things, verbatim, in every project:

1. `os.MkdirAll`s the logs directory, falling back to `os.TempDir()/logs` when the
   passed dir is empty;
2. wires `CyclicOverwritingFilesHandler(folder, name, 5*1024*1024, 7)`;
3. registers a private `printer` hook for every level `>= printerMinLevel`. The
   `printer` is an `io.Writer` that prefixes a `"2006/01/02 15:04:05"` timestamp and
   indents continuation lines of multi-line messages, writing to **stdout** — i.e. a
   timestamped, multiline-aware sibling of the library's `PrintHandler()`;
4. redirects the standard library `log` package via
   `log.SetOutput(l.WriterAs(<warning-level>))` + `log.SetFlags(0)`.

The boilerplate has already rotted in transit: one consumer pasted a dead
`Logger`+`SetTelegramHandler` wrapper and a Telegram label belonging to a different
project. Copy-paste is doing exactly what copy-paste always does.

This plan extracts that boilerplate **into the library** as two purely **additive**
helpers, and nothing else:

- `TimestampedPrintHandler(...)` in `handler.go` — a handler factory, sibling to
  `PrintHandler()`, returning the package's mutex-guarded `io.Writer`. Timestamp prefix
  + multiline continuation indent, to stdout.
- `NewFileLogger(folder, prefix string, minLevel LogLevel, opts ...FileLoggerOption) (*Logger, error)`
  — a convenience constructor in a new `filelogger.go` that absorbs dir creation
  (with an **opt-in** temp fallback), the cyclic file-handler wiring, an attached
  timestamped stdout printer, and an **opt-in** std-`log` redirect.

The two design landmines from the legacy code — the global `log` mutation and the
silent temp-dir fallback — are NOT reproduced as silent defaults. The `log` redirect is
opt-in; the temp fallback is opt-in. Both are surfaced under Risks and Trade-offs.

### Hard constraints

- **Additive only. Backward compatible. New minor version (e.g. v1.x → v1.(x+1)).**
- Do **NOT** change `NewLogger`'s existing signature (`NewLogger(min LogLevel, handlers ...io.Writer)`).
- Do **NOT** change `PrintHandler`, `CyclicOverwritingFilesHandler`, the `writer` type,
  `WriterAs`, `Hook`, or any other existing exported or unexported symbol's behaviour.
- No CGO; no new dependencies (`stretchr/testify` for tests, `twinj/uuid` already
  present — nothing new).

### Dependency on plan 003 (`levels` preset)

Plan 003 introduces a shared `levels` preset (DEBUG/INFO/WARNING/SEVERE-style
constants). **It does not exist on disk yet** (verified: no `levels.go`, no exported
level constants in the package; the only level constants live in `logger_test.go` as
unexported test fixtures). This plan therefore treats 003 as a **soft** dependency:

- `NewFileLogger` takes `minLevel` as an explicit caller-supplied argument — it never
  depends on a preset constant existing.
- The std-`log` redirect needs *a* level to forward at. Rather than hard-wire a magic
  number or import a not-yet-existent constant, the redirect level is itself
  configurable via an option (see Task 3), defaulting to the same `minLevel` the caller
  already passed. If/when 003 lands, callers can pass `levels.Warning` (or whatever it
  is named) — no API change here is needed.

This plan does **not** block on 003 and 003 does **not** block on this plan.

## Assumptions

- "Continuation indent" means: the first physical line of a message is written as
  `<timestamp> <line>`, and every subsequent `\n`-separated line is written indented so
  it visually aligns under the message body rather than the timestamp. The exact indent
  is a fixed-width pad equal to the rendered timestamp + separator width (so multi-line
  payloads read as a block). This matches the observed legacy `printer` behaviour. If
  the user wants a different indent rule, that is a one-line change in Task 1; flagged
  in Risks.
- The timestamp layout default is `"2006/01/02 15:04:05"` (matching the legacy code and
  the std `log` default-ish format), overridable via option.
- "to stdout" means `os.Stdout`. The existing `PrintHandler` uses the builtin
  `println`, which writes to **stderr** — `TimestampedPrintHandler` deliberately differs
  (stdout) to match the legacy `printer`. This divergence is intentional and documented
  in the godoc; it is NOT a bug to "fix" in `PrintHandler`. (See Risks.)
- The cyclic-file defaults baked into `NewFileLogger` are the legacy values:
  `maxFileCapacity = 5 * 1024 * 1024` (5 MiB) and `maxFilesInFolder = 7`. Both are
  overridable via options so the helper does not become a straitjacket.
- The printer is attached as a **hook over every level `>= minLevel`**, reproducing the
  legacy fan-out semantics (printer sees a level even if it is below the handler's
  minimum? — no: the legacy printer was registered only for levels `>= printerMinLevel`,
  so it mirrors handler gating, not the hook-fires-below-minimum asymmetry). To replicate
  exactly, the printer is attached as a **handler**, not a hook — handlers fire on
  `level >= minimumLogLevel`, which is precisely the legacy "every level >= minLevel"
  behaviour and avoids enumerating an open-ended level range into `Hook(...)`. See
  Task 2 / Pitfalls for why handler-attachment is the faithful translation.
- `LogLevel` remains consumer-defined; this plan introduces no level constants.
- New code lives in `filelogger.go` (constructor + options) and `handler.go` (the new
  handler factory), with tests in `filelogger_test.go` and additions to
  `handler_test.go`. No existing file's existing functions are edited.

## Tasks

### Task 1: Add `TimestampedPrintHandler` factory to `handler.go`

- **Description:**
  Add a new exported handler factory `TimestampedPrintHandler(opts ...PrintOption) io.Writer`
  in `handler.go`, immediately after `PrintHandler()` (it is its sibling). It returns
  the package's mutex-guarded `*writer` (same pattern as every other handler in this
  file: `&writer{ h: func(msg []byte) (int, error) { ... } }`). The handler:
  1. renders a timestamp prefix using a layout (default `"2006/01/02 15:04:05"`);
  2. writes the first line of `msg` as `<timestamp> <firstline>`;
  3. for multi-line messages, writes each subsequent line indented by a pad whose width
     equals `len(renderedTimestamp) + len(separator)` so the body aligns as a block;
  4. trims the trailing newline/space of the incoming message before splitting (mirror
     `PrintHandler`'s `bytes.TrimSpace`), then emits exactly one trailing `\n`;
  5. writes to `os.Stdout` (NOT the builtin `println`, which targets stderr);
  6. returns `(len(msg), nil)` on success, or `(bytesWritten, err)` if the stdout write
     fails — do not swallow the error (`fmt.Fprint*` to a writer is the documented
     exception, but here we own the write and should surface a real I/O failure).

  Define a small, minimal options surface so the factory stays consistent with the
  other zero-or-few-arg factories in this file:

  ```go
  // PrintOption configures TimestampedPrintHandler.
  type PrintOption func(*printConfig)

  type printConfig struct {
      layout    string    // time layout; default "2006/01/02 15:04:05"
      out       io.Writer // sink; default os.Stdout
      clock     func() time.Time // injected for tests; default time.Now
  }

  // WithTimeLayout overrides the timestamp layout used by TimestampedPrintHandler.
  func WithTimeLayout(layout string) PrintOption { ... }

  // WithOutput overrides the sink (default os.Stdout). Primarily for testing.
  func WithOutput(w io.Writer) PrintOption { ... }
  ```

  `clock` is an unexported test seam set by an unexported option used only in tests
  (see Pitfalls — do NOT expose a clock-injection option in the public API just for
  tests; keep it package-private). `WithOutput` is public because redirecting the sink
  is a legitimate consumer need (and is what `NewFileLogger`'s tests will use).

- **Acceptance Criteria:**
  - [ ] `TimestampedPrintHandler()` exists in `handler.go`, returns `io.Writer`, and
        is the package's `*writer` (mutex-guarded) — verified by the same structural
        pattern as the other factories.
  - [ ] Single-line input `"hello"` produces output `"<ts> hello\n"` for the configured
        layout.
  - [ ] Multi-line input `"line1\nline2\nline3"` produces `"<ts> line1\n"` followed by
        `"line2"` and `"line3"` each indented by exactly `len("<ts> ")` spaces, and one
        trailing `\n` overall — asserted byte-for-byte using `WithOutput(&bytes.Buffer{})`
        and a fixed `clock`.
  - [ ] Trailing whitespace/newline on input is trimmed before formatting (input
        `"hi\n"` and `"hi"` produce identical output).
  - [ ] Empty/whitespace-only input produces a single `"<ts>\n"` (or `"<ts> \n"` — pick
        one and assert it; document the choice) and does not panic.
  - [ ] An error from the underlying sink is returned, not swallowed (assert with a
        sink whose `Write` returns an error).
  - [ ] Exported identifiers (`TimestampedPrintHandler`, `PrintOption`, `WithTimeLayout`,
        `WithOutput`) carry godoc starting with the identifier and ending with a period;
        the godoc documents the stdout default and the intentional difference from
        `PrintHandler` (stdout vs stderr).
  - [ ] `gofmt -l .` empty; `go vet ./...` clean.

- **Pitfalls & edge cases:**
  - The default `out` must be resolved **lazily inside the closure or at construction**,
    not captured as a nil interface. Resolve `os.Stdout` in the config defaults before
    building the `writer`.
  - Do NOT expose `clock` injection publicly. Tests need deterministic timestamps;
    achieve that with an unexported `withClock(func() time.Time)` option in the test
    file's package (tests are in-package `loginjector`, so they can call unexported
    helpers). Keep the public surface to `WithTimeLayout` + `WithOutput`.
  - Indent width must be computed from the **rendered** timestamp length, not the layout
    string length — `len("2006/01/02 15:04:05")` happens to equal the rendered width for
    that layout, but a custom layout (e.g. with a timezone) would break a layout-length
    assumption. Compute `len(ts) + len(sep)`.
  - Splitting on `\n`: a message ending in `\n` after `TrimSpace` will not produce a
    trailing empty line — good, but a message with an *internal* blank line
    (`"a\n\nb"`) must keep the blank line (indented to empty). Assert this.
  - Do not use the builtin `println` (stderr, not configurable). Use
    `fmt.Fprint`/`io.WriteString` to the resolved `out`.
  - Concurrency: the surrounding `*writer` already serialises calls, so the closure
    needs no extra locking. Do not add a second mutex.

- **Complexity:** Medium

- **Code Example:**

  ```go
  // TimestampedPrintHandler prints each message to stdout with a leading timestamp,
  // indenting the continuation lines of a multi-line message so the body reads as a
  // block. Unlike PrintHandler, which writes to stderr via the builtin println, this
  // handler writes to os.Stdout (overridable with WithOutput). The returned writer is
  // mutex-guarded; the logger never calls its Write concurrently.
  func TimestampedPrintHandler(opts ...PrintOption) io.Writer {
      cfg := printConfig{layout: "2006/01/02 15:04:05", out: os.Stdout, clock: time.Now}
      for _, o := range opts {
          o(&cfg)
      }
      w := &writer{
          h: func(msg []byte) (int, error) {
              ts := cfg.clock().Format(cfg.layout)
              const sep = " "
              indent := strings.Repeat(" ", len(ts)+len(sep))

              body := string(bytes.TrimSpace(msg))
              lines := strings.Split(body, "\n")

              var b strings.Builder
              b.WriteString(ts)
              b.WriteString(sep)
              b.WriteString(lines[0])
              b.WriteByte('\n')
              for _, ln := range lines[1:] {
                  b.WriteString(indent)
                  b.WriteString(ln)
                  b.WriteByte('\n')
              }
              if _, err := io.WriteString(cfg.out, b.String()); err != nil {
                  return 0, err
              }
              return len(msg), nil
          },
      }
      return w
  }
  ```

  (Note: this snippet illustrates the approach; the implementer should finalise the
  empty-input behaviour and the `len(msg)` vs bytes-written return per the acceptance
  criteria.)

### Task 2: Add `NewFileLogger` constructor + options in a new `filelogger.go`

- **Description:**
  Create a new file `filelogger.go` (package `loginjector`) containing the convenience
  constructor and its functional-options type. Signature:

  ```go
  func NewFileLogger(folder, prefix string, minLevel LogLevel, opts ...FileLoggerOption) (*Logger, error)
  ```

  Behaviour, in order:
  1. Apply options to a config struct seeded with the legacy defaults:
     `maxFileCapacity = 5 << 20`, `maxFilesInFolder = 7`, temp-fallback **off**,
     std-`log` redirect **off**, printer **on** (it is the whole point), printer layout
     default, redirect level defaults to `minLevel`.
  2. Resolve the target folder:
     - if `folder != ""`, use it as-is;
     - if `folder == ""` AND the temp-fallback option is enabled, use
       `filepath.Join(os.TempDir(), "logs")`;
     - if `folder == ""` AND the fallback is disabled, return a clear error
       (`fmt.Errorf("loginjector: empty folder and temp fallback not enabled")`).
       Do NOT silently invent a path.
  3. `os.MkdirAll(folder, 0750)` (dir perms — note files are `0640` via
     `defaultFilePermissions`; dirs need the execute bit, hence `0750`, not `0640`).
     Return the wrapped error on failure.
  4. Build `CyclicOverwritingFilesHandler(folder, prefix, cfg.maxFileCapacity, cfg.maxFilesInFolder)`.
  5. Build the printer (`TimestampedPrintHandler(...)` from Task 1) if the printer is
     enabled, applying any printer options forwarded through `FileLoggerOption`s.
  6. `l, err := NewLogger(minLevel, <fileHandler>, <printer?>)` — pass the printer as a
     **handler**, so it fires on `level >= minLevel`, faithfully reproducing the legacy
     "printer for every level >= printerMinLevel" semantics without enumerating an
     open-ended level set into `Hook(...)`. (See Pitfalls for why this is correct and
     not the hook path.)
  7. If the std-`log` redirect option is enabled, call
     `log.SetOutput(l.WriterAs(cfg.redirectLevel))` and `log.SetFlags(0)`. This is the
     only place global state is touched, and only when explicitly opted in.
  8. Return `(l, nil)`.

  Options to define (functional options, consistent with Task 1's style):

  ```go
  type FileLoggerOption func(*fileLoggerConfig)

  func WithMaxFileCapacity(bytes uint32) FileLoggerOption
  func WithMaxFilesInFolder(n int) FileLoggerOption
  func WithTempDirFallback() FileLoggerOption          // opt-in; off by default
  func WithStdLogRedirect(level LogLevel) FileLoggerOption // opt-in; off by default
  func WithoutPrinter() FileLoggerOption                // disable the stdout printer
  func WithPrinterOptions(opts ...PrintOption) FileLoggerOption // forward to Task 1
  ```

  Naming `WithStdLogRedirect(level)` carries the level so the caller is forced to be
  explicit about what severity std-`log` lines map to (the legacy code hard-coded a
  warning level; we make it a conscious choice). If a future `levels` preset (plan 003)
  lands, callers pass `levels.Warning` here.

- **Acceptance Criteria:**
  - [ ] `NewFileLogger` exists in `filelogger.go`, compiles, and returns
        `(*Logger, error)`.
  - [ ] A logger created with a real temp folder writes a message and the rotated file
        `prefix.00000001.log` exists and contains the message (assert file contents).
  - [ ] Writing enough bytes to exceed `WithMaxFileCapacity(small)` rolls over to
        `prefix.00000002.log` (assert second file appears) — reuse the rotation idiom
        already covered by the existing `CyclicOverwritingFilesHandler` tests.
  - [ ] With `folder == ""` and `WithTempDirFallback()`, the logger is created and the
        log file lands under `os.TempDir()/logs` (assert the resolved path; clean up in
        `t.Cleanup`).
  - [ ] With `folder == ""` and **no** fallback option, `NewFileLogger` returns a
        non-nil error and a nil `*Logger`.
  - [ ] **By default (no `WithStdLogRedirect`), the global `log` output is NOT changed.**
        Asserted by capturing `log`'s state and confirming `NewFileLogger(...)` without
        the option leaves a sentinel write to the std `log` package going to the
        test-controlled previous output, not to the new logger. (See Pitfalls for the
        capture technique.)
  - [ ] With `WithStdLogRedirect(level)`, a `log.Print("x")` is observed flowing through
        the logger at `level` (assert via a `WithOutput`-redirected printer or a probe
        handler), and `log.Flags() == 0` afterwards.
  - [ ] `WithoutPrinter()` produces a logger whose only handler is the file handler
        (no stdout output) — assert no stdout printer side effect.
  - [ ] All exported identifiers carry conforming godoc; the `NewFileLogger` godoc
        explicitly documents that the std-`log` redirect mutates global process state
        and is opt-in, and that the temp fallback is opt-in.
  - [ ] `CGO_ENABLED=0 go test -race ./...` green; `go vet ./...` clean; `gofmt -l .`
        empty.

- **Pitfalls & edge cases:**
  - **Printer as handler, not hook.** The legacy printer fired for "every level
    `>= printerMinLevel`". The library's *hooks* fire on **exact** level match
    (`level == hook.Level`) and, critically, fire *below* `minimumLogLevel` — the wrong
    semantics. Attaching the printer as a normal **handler** to `NewLogger` gives exactly
    `level >= minLevel` gating, which is what the legacy code meant. Do not loop over a
    guessed range of levels calling `Hook(...)`; that both leaks the printer below the
    minimum and assumes a finite level set. This is the single most important
    correctness decision in the task.
  - **Dir perms.** Files use `defaultFilePermissions = 0640`. Directories must be
    `0750` (need the execute/traverse bit). Reusing `0640` for `MkdirAll` yields a dir
    you cannot enter. Do not import `defaultFilePermissions` for the dir.
  - **Global `log` capture in tests.** `log.SetOutput`/`log.SetFlags` are process-global.
    Tests touching them must snapshot and restore (`old := log.Writer()` is not exported
    pre-1.21; use `log.Default().Writer()` on 1.22, restore in `t.Cleanup`) and must NOT
    run `t.Parallel()` against each other — a parallel test that doesn't touch `log`
    could observe a redirected global. Mark the redirect tests serial; document why.
  - **Temp-fallback path leakage.** Returning `os.TempDir()/logs` writes outside the
    caller's intended tree. That is exactly why it is opt-in. The error path for empty
    folder + no fallback must be tested so the "no silent invention" guarantee is
    enforced.
  - **`maxFileCapacity` type.** `CyclicOverwritingFilesHandler` takes `uint32`. The
    option must take/clamp to `uint32`; `5 << 20` fits. Don't widen the underlying
    handler's type.
  - **Empty prefix.** `CyclicOverwritingFilesHandler` will happily build `.00000001.log`
    with an empty prefix (filename starts with a dot). Decide: either allow it (document)
    or reject empty prefix in `NewFileLogger`. Recommend rejecting empty `prefix` with a
    clear error — a hidden dotfile log is a footgun. Assert the rejection.
  - **Cleanup.** Every test creating files must use `t.TempDir()` (auto-cleaned) where
    possible; the temp-fallback test, which writes to the *real* `os.TempDir()/logs`,
    must remove only the files it created in `t.Cleanup` and must not nuke a shared dir
    other processes might use. Prefer a unique prefix per test run.

- **Complexity:** Hard

- **Code Example:**

  ```go
  // NewFileLogger builds a Logger that writes to size-rotated files under folder and
  // mirrors every message at or above minLevel to a timestamped stdout printer.
  //
  // The standard library log package is NOT redirected unless WithStdLogRedirect is
  // passed — that redirect mutates global process state. An empty folder is rejected
  // unless WithTempDirFallback is passed, in which case logs land under
  // os.TempDir()/logs.
  func NewFileLogger(folder, prefix string, minLevel LogLevel, opts ...FileLoggerOption) (*Logger, error) {
      cfg := fileLoggerConfig{
          maxFileCapacity:  5 << 20,
          maxFilesInFolder: 7,
          printer:          true,
          redirectLevel:    minLevel,
      }
      for _, o := range opts {
          o(&cfg)
      }

      if prefix == "" {
          return nil, fmt.Errorf("loginjector: file logger prefix must not be empty")
      }

      if folder == "" {
          if !cfg.tempFallback {
              return nil, fmt.Errorf("loginjector: empty folder and temp fallback not enabled")
          }
          folder = filepath.Join(os.TempDir(), "logs")
      }
      if err := os.MkdirAll(folder, 0750); err != nil {
          return nil, fmt.Errorf("loginjector: create log folder %q: %w", folder, err)
      }

      handlers := []io.Writer{
          CyclicOverwritingFilesHandler(folder, prefix, cfg.maxFileCapacity, cfg.maxFilesInFolder),
      }
      if cfg.printer {
          handlers = append(handlers, TimestampedPrintHandler(cfg.printerOpts...))
      }

      l, err := NewLogger(minLevel, handlers...)
      if err != nil {
          return nil, err
      }

      if cfg.stdLogRedirect {
          log.SetOutput(l.WriterAs(cfg.redirectLevel))
          log.SetFlags(0)
      }
      return l, nil
  }
  ```

## Risks

- **Global `log` mutation (the loud one).** `log.SetOutput` / `log.SetFlags` change
  process-wide state shared by every package in the consuming binary. Even gated behind
  `WithStdLogRedirect`, a consumer that opts in is silently re-routing every other
  library's `log.Print`. Mitigation: opt-in only, named explicitly with a required
  level argument, and documented in the `NewFileLogger` godoc as a global side effect.
  The library itself never calls it in any default path.
- **Surprising temp-fallback path.** `os.TempDir()/logs` is outside the caller's tree
  and world-readable-ish on some systems; logs landing there silently is a data-leak /
  "where are my logs?" footgun. Mitigation: opt-in (`WithTempDirFallback`), and a hard
  error on empty folder when the option is absent — no silent path invention.
- **stdout vs stderr divergence from `PrintHandler`.** `PrintHandler` writes to stderr
  (builtin `println`); `TimestampedPrintHandler` writes to stdout (to match the legacy
  printer). A reader who assumes the two are stream-identical will be surprised.
  Mitigation: documented in godoc; `WithOutput` lets a caller realign them. We do NOT
  change `PrintHandler`.
- **Indent semantics are a guess from copy-pasted code.** The exact continuation-indent
  rule (pad = timestamp+sep width) is inferred. If the real intent differs, it is a
  one-line change in Task 1, but the byte-exact tests would need updating. Low blast
  radius, flagged so the user can confirm the format before sign-off.
- **Test pollution via global `log`.** Tests touching `log` can corrupt sibling tests if
  not serialised + restored. Mitigation: explicit serial ordering and `t.Cleanup`
  restoration; called out in Task 3.
- **Version bump discipline.** Additive API still requires a **minor** version tag so
  downstream `go get -u` picks it up cleanly. Forgetting the tag leaves consumers unable
  to drop their boilerplate. Mitigation: note the tag in the release step (out of scope
  for code, in scope for the PR description).

## Trade-offs

- **std-`log` redirect: opt-in over default.** Chose an explicit `WithStdLogRedirect(level)`
  over reproducing the legacy always-on redirect, because mutating global process state
  as a hidden side effect of "make a logger" violates least-surprise and would make the
  helper unsafe to call more than once / in libraries. Cost: consumers migrating from
  the boilerplate must add one option call to keep old behaviour. Worth it — the old
  behaviour was a latent bug, not a feature.
- **Temp fallback: opt-in over default.** Chose to reject empty folder by default and
  gate the `os.TempDir()/logs` fallback behind `WithTempDirFallback`, over reproducing
  the legacy silent fallback. Rejected silent fallback because "logs vanished into
  /tmp" is a recurring real-world support cost. Cost: one more option for consumers who
  actually wanted the fallback. Acceptable.
- **Printer as handler, not hook.** Chose to attach the timestamped printer as a normal
  handler (`level >= minLevel`) rather than enumerate levels into `Hook(...)`. Chosen
  because it (a) exactly matches the legacy "every level >= printerMinLevel" intent,
  (b) avoids the hook's fire-below-minimum asymmetry, and (c) does not assume a finite
  level set (`LogLevel` is an open `int`). Cost: the printer cannot be detached
  independently of the file handler via `Unhook` — acceptable, since the helper owns the
  whole logger and a consumer wanting fine-grained control can build the logger by hand
  with `NewLogger`.
- **Functional options over a config struct or many positional args.** Chose
  `opts ...FileLoggerOption` / `opts ...PrintOption` to keep the common case
  (`NewFileLogger(dir, "app", min)`) clean while leaving room for capacity/retention/
  redirect/printer tuning. Consistent with keeping the existing factories' minimal
  positional signatures untouched. Cost: slightly more surface area than a single struct,
  but matches Go idiom and the rest of this package's "factory returns io.Writer" style.
- **New `filelogger.go` over folding into `logger.go`/`handler.go`.** Chose a dedicated
  file to keep the convenience layer (which has opinions: defaults, global side effects)
  cleanly separable from the primitive `Logger`/handler core. Cost: one more file;
  benefit: the opinionated bootstrap is quarantined from the unopinionated core.
- **Keep `minLevel` an explicit argument, no preset coupling.** Chose to not import or
  require plan 003's `levels` preset so this helper ships independently and stays usable
  by consumers with their own level schemes. Cost: callers without a preset pass a raw
  `LogLevel`; with 003 they pass a named constant. No lock-in either way.
